Understanding MapleStory WZ Files

A practical intro to MapleStory's data archives.

Published June 2026 · All blog posts

If you've ever dug into MapleStory modding, private servers, or sprite ripping, you've heard of WZ files. They're the packed archives that hold most of the game's graphics, sounds, and configuration data. This guide explains what they are, how they're organized, and how you can explore them without uploading your game files to random websites.

What Is a WZ File?

WZ is a proprietary archive format used by MapleStory (and some other Nexon titles) to bundle game assets efficiently. Instead of thousands of loose PNG and XML files on disk, the client stores them in versioned archives like Character.wz, Map.wz, Item.wz, and Mob.wz.

Each archive contains a tree of nodes. Leaf nodes might be images, sound clips, or XML-like property data describing stats, animations, and map objects. Parent nodes group related content - for example, a folder for a specific hair style or a map tileset.

Common WZ Archives

Your MapleStory installation folder typically includes archives such as:

  • Character.wz - player appearances, equipment sprites, animations
  • Item.wz - item icons and related graphics
  • Map.wz - map backgrounds, objects, and tile data
  • Mob.wz - monster sprites and animation data
  • Skill.wz - skill effects and icons
  • Sound.wz - music and sound effects

Exact file names and encryption vary by client version and region. Private server communities often maintain version-specific tooling for this reason.

How the Tree Structure Works

Think of a WZ file like a filesystem inside a single binary package. When you open one, you see a hierarchy:

  1. Top-level categories (e.g., categories of items or maps)
  2. Subfolders for specific IDs or names
  3. Leaf nodes containing actual data - PNG images, canvas instructions, or XML properties

Property nodes often store numbers and strings - damage values, map spawn points, string labels. Image nodes store compressed bitmap data that tools decode into PNG or render to a canvas.

Encryption & Version Changes

Nexon has updated WZ encryption schemes over the years. A decoder that works for one client version may fail on another. Symptoms include garbled tree names, failed image decode, or empty nodes. This is why WZ tooling needs ongoing maintenance after major patches.

If you're working with files from your own legitimate installation, always note the client version when reporting decode issues.

Exploring WZ Files Safely

Historically, players used desktop programs to open WZ archives. That works, but many online "converters" ask you to upload game files - something you should avoid with proprietary client data.

I built wzbrowser to solve this differently: load a WZ file from your computer, decode it entirely in the browser with JavaScript, browse the tree, preview images on a canvas, and export XML or PNG locally. Nothing gets uploaded.

Typical Workflow with wzbrowser

  1. Locate the WZ file in your MapleStory installation directory
  2. Open wzbrowser and load the file
  3. Browse the node tree to find the asset category you need
  4. Preview sprites on the canvas to confirm you have the right node
  5. Export PNG or XML for your personal reference or project

Legal & Ethical Considerations

MapleStory assets are copyrighted. Extracting files for personal learning or reference is one thing; redistributing sprites, running unauthorized services, or selling extracted content is another. Always read Nexon's terms of service and respect intellectual property.

This guide and wzbrowser are fan-made tools for education and personal use. They are not affiliated with Nexon.

Troubleshooting

  • Tree looks empty - wrong encryption for your client version; try an updated decoder
  • Images look corrupted - node may use a format variant; check related nodes or version-specific docs
  • Large files load slowly - normal for big archives; let the browser finish parsing
